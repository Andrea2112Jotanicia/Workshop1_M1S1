> Daily Sales Recording System.

>> Start of the Program.
- The file from which the program is executed is the main.py file.

- The first thing it shows is a Welcome message.

>> Sales Record.

- Then the system starts the sales registration process.

- The program prompts the user to enter the following information:

* Product Name
* Unit Price
* Quantity Sold

>> Data Validation.

All data entered by the user undergoes a validation process using specific methods.

The system verifies that:

- Text fields contain only string values.

- Numeric fields requiring decimal values ​​contain float values.

- Fields requiring whole numbers contain integer values.

>> Data Storage.

- Once the entered data has been successfully validated, the sales information is stored within a list.

>> Program Control.

- After registering each sale, the system asks the user to confirm if they wish to continue registering new sales.

- Depending on the user's response, the control variable changes its value and determines whether the loop continues or the program advances to the next stage.

>> Data Processing.

At this stage, the program calculates:

- The total quantity sold for each product.

- The total revenue generated during the day.

>> Generation of the Final Summary.

After processing the information, the system generates a final report.
This report shows:

- The list of registered products.

- The total quantity sold for each product.

- The total amount of money collected.

>> Ending of program.
